# Fix Google OAuth "Error 400: invalid_request"

## The Problem

You're getting "Error 400: invalid_request" because the redirect URI generated by your app doesn't match what's configured in Google Cloud Console.

## Solution: Add Redirect URI to Google Cloud Console

### Step 1: Find Your Redirect URI

When you run the app and try to sign in, check the console logs. You'll see:
```
🔗 Redirect URI: https://auth.expo.io/@your-username/buddia
```

**Copy this exact URI** - it will look something like:
- `https://auth.expo.io/@your-expo-username/buddia`

### Step 2: Add to Google Cloud Console

1. Go to [Google Cloud Console](https://console.cloud.google.com)
2. Select your project: **buddia-dev**
3. Navigate to: **APIs & Services** → **Credentials**
4. Find your OAuth 2.0 Client ID: `64676977478-1lq4fnf0khpmhls1qcodbotrpa26md8m`
5. Click on it to edit
6. Under **"Authorized redirect URIs"**, click **"+ ADD URI"**
7. Paste the redirect URI from Step 1 (e.g., `https://auth.expo.io/@your-username/buddia`)
8. Click **"SAVE"**

### Step 3: Verify OAuth Consent Screen

Also make sure your OAuth consent screen is configured:

1. In Google Cloud Console, go to: **APIs & Services** → **OAuth consent screen**
2. Make sure:
   - **User Type**: Internal or External (depending on your needs)
   - **App name**: buddia-dev (or your app name)
   - **Support email**: Your email
   - **Scopes**: Should include `openid`, `profile`, `email`
   - **Test users** (if in Testing mode): Add your Google account email

### Step 4: Wait a Few Minutes

After saving, wait 2-3 minutes for Google's servers to update, then try again.

## Alternative: Use Custom Scheme (Production)

If you want to use your custom scheme `buddi://` instead of the proxy:

1. Change in code:
   ```typescript
   const redirectUri = AuthSession.makeRedirectUri({
     useProxy: false, // Use custom scheme
   });
   ```

2. Add to Google Cloud Console:
   - Add `buddi://` as an authorized redirect URI
   - Note: This requires rebuilding your app (not just updating)

## Debugging

If it still doesn't work:

1. Check the console log for the exact redirect URI
2. Verify it matches EXACTLY in Google Cloud Console (including trailing slashes, etc.)
3. Make sure you're using the correct OAuth Client ID (Web client, not iOS/Android)
4. Check that the OAuth consent screen is published (if using External user type)

## Common Issues

- **"Redirect URI mismatch"**: The URI in code doesn't match Google Cloud Console
- **"Invalid client"**: Wrong Client ID or Client ID for wrong platform
- **"Access blocked"**: OAuth consent screen not configured or app in testing mode without test users
